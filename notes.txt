
Notes:

  * Create a symbolic link

    Open Command Prompt as an Administrator.
    Create the symbolic link using the "mklink /D" command. 
    The format is: mklink /D <Link_Path> <Target_Path>
    mklink /D "C:\Users\**USER_NAME**\AppData\Roaming\QGIS\QGIS3\profiles\default\python\plugins\**PLUGIN_NAME**" "C:\**DEV_DIRECTORY**\**PLUGIN_NAME**"


  * Compile the resources file using pyrcc5

	  use "cd PATH_TO_PLUGIN" in the OSGeo4W Shell and execute "compile.bat"


  * You can use the Makefile to compile your Ui and resource files when
    you make changes. This requires GNU make (gmake)


  * Translation Workflow
    
    1. Compile the Translation by using compile.bat found in the root directory in the OSGeo4W Shell. 
    The pylupdate5 command inside the script will find all your new strings and add them to i18n/de.ts. 

    2. Open the Qt Linguist application by usint the command "linguist" in a OSGeo4W Shell. 
    In Linguist, open the de.ts file in the i18n directory. 
    For each English string, you can enter the corresponding German translation. Save your changes in Linguist.
    
    3. Run the compile.bat file again
    The lrelease command in the script will take your updated de.ts file and create the final i18n/de.qm file.
