
Notes:

  * Create a symbolic link

    Open Command Prompt as an Administrator.
    Create the symbolic link using the "mklink /D" command. 
    The format is: mklink /D <Link_Path> <Target_Path>

    mklink /D "C:\Users\**USER_NAME**\AppData\Roaming\QGIS\QGIS**X**\profiles\default\python\plugins\**PLUGIN_NAME**_dev" "C:\**DEV_DIRECTORY**\**PLUGIN_NAME**"


  * Translation Workflow
    
    1. Compile the Translation by using compile.bat found in the root directory in the OSGeo4W Shell. 
    This will find all new strings and add them to i18n/de.ts. 

    2. Open the Qt Linguist application by usint the command "linguist" in a OSGeo4W Shell. 
    In Linguist, open the de.ts file in the i18n directory. 
    For each English string, you can enter the corresponding translation. Save your changes in Linguist.
    
    3. Run the compile.bat file again
    The lrelease command in the script will take the de.ts file and create the final i18n/de.qm file.
    If lrelease is not installed, go to OSGeo4W setup and install "qt6-devel" (in the "Libs" section)


  * Publishing New Versions (Automated)

    1. Update the `version` number and changelog in `metadata.txt`. 

    2. Run the release script from the OSGeo4W Shell in the plugin directory using the following commands:
       
      `cd **PATH_TO_PLUGIN**`
      `python release.py`
       
      This script will update `plugins.xml`, compile resources, and package the new .zip file.
      The .zip file and plugins.xml will be saved to the shared repo (see `download_url_base` in metadata.txt).
